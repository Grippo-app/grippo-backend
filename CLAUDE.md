# grippo-backend

## Project purpose

REST API for Grippo. Auth (email + Google + Apple), users/profiles,
trainings, exercise examples + metrics, dictionaries (muscles/equipments),
weight history, push tokens, admin endpoints. Single source of truth for
all Grippo data. Deployed via Docker Compose (NestJS + PostgreSQL).

---

## Stack

- Node.js **20** (Docker runtime), TypeScript **5.1**.
- NestJS **10** + `@nestjs/platform-express`, `@nestjs/config`, `@nestjs/swagger`, `@nestjs/jwt`, `@nestjs/passport`.
- PostgreSQL **15.5**.
- TypeORM **0.3** with custom `SnakeNamingStrategy`. Schema is managed via `synchronize` (env-driven), no migrations (`migrations: []`).
- Auth: JWT access + refresh, `passport-jwt`, `bcryptjs`, `google-auth-library` (Google ID tokens), Apple (custom `services/apple-auth.service` via authorization code).
- Validation: `class-validator` + `class-transformer`, global `ValidationPipe({ transform: true, whitelist: true, forbidNonWhitelisted: true })`.
- Logging: custom `LoggingInterceptor` (method/url/status/uid/duration, body for POST/PUT) + `morgan`.
- HTTP id: `uuid` v4 (generated by hand in services via `v4()`, **not** `@PrimaryGeneratedColumn` to control transactions).
- Time: `moment` (validation of date-range parameters).
- Swagger: `setupSwagger(app)`, available at `/docs`. JSON schema — `/docs-json`. Bearer scheme `'access-token'`.
- Deploy: Docker Compose (`db` + `backend`), multi-stage Dockerfile (`node:20` builder → `node:20-slim` runtime, non-root user). Healthcheck — HTTP `/health`.

## Bootstrap order (`src/main.ts`)

```typescript
async function bootstrap() {
  const app = await NestFactory.create<NestExpressApplication>(
    AppModule, new ExpressAdapter(), { cors: true },
  );

  // 1. Global ValidationPipe — strict whitelist
  app.useGlobalPipes(new ValidationPipe({
    transform: true,
    whitelist: true,
    forbidNonWhitelisted: true,
  }));

  // 2. Global LoggingInterceptor
  app.useGlobalInterceptors(new LoggingInterceptor());

  // 3. Swagger setup (reads all @ApiProperty/@ApiOperation)
  setupSwagger(app);

  // 4. Bind to 0.0.0.0:${BACKEND_PORT}
  await app.listen(port, '0.0.0.0');

  // 5. Graceful shutdown on SIGINT/SIGTERM
}
```

`AppModule` (`src/app.module.ts`) wires up:

- **Shared**: `ConfigModule.forRoot({ envFilePath: '.env', isGlobal: true })`, `DatabaseModule`, `I18nModule` (with `@Global()` decorator → available everywhere without explicit import).
- **Features**: `AuthModule`, `UsersModule`, `TrainingsModule`, `ExerciseExampleModule`, `MusclesModule`, `WeightHistoryModule`, `ExerciseMetricsModule`, `EquipmentsModule`, `AdminModule`, `PushTokensModule`.

## Module layout

### Feature module structure

`src/modules/<feature>/`:

```
<feature>.controller.ts   // HTTP handlers + Swagger decorators + @UseGuards(JwtAuthGuard)
<feature>.service.ts      // Business logic + TypeORM repositories
<feature>.module.ts       // @Module — imports DatabaseModule, providers includes ...repositoryProviders
dto/
  <name>.request.ts       // Input DTO + class-validator decorators
  <name>.response.ts      // Output DTO + @ApiProperty
  <name>.dto.ts           // Optional shared DTOs inside the feature
services/                 // (optional) extracted helpers — TokenService, GoogleAuthService, AppleAuthService
```

Every feature `*.module.ts`:

```typescript
@Module({
  imports: [DatabaseModule],
  providers: [TrainingsService, ...repositoryProviders],
  controllers: [TrainingsController],
  exports: [TrainingsService],
})
export class TrainingsModule {}
```

Services are exported → `AdminModule` imports `TrainingsModule`/`UsersModule`/etc. and reuses their services inside `AdminController`.

### Shared layout

- `src/main.ts` — bootstrap. See section above.
- `src/app.module.ts` — single root module.
- `src/swagger.ts` — `DocumentBuilder` with Bearer scheme `'access-token'`, custom `swagger-custom.js` adding a "Swagger JSON" link to the topbar.
- `src/entities/*.entity.ts` — all TypeORM entities **flat**, no subdirectories. File — kebab-case (`user-profiles.entity.ts`), class — `<Name>Entity`, `@Entity({ name: 'snake_case_table' })`.
- `src/common/`:
  - `jwt.strategy.ts` — `JwtStrategy extends PassportStrategy(Strategy)`. Fetches user by `payload.id` via `USERS_REPOSITORY`. Throws plain `Error('TokenExpiredError')` / `Error('JsonWebTokenError')` — not NestJS exceptions; `JwtAuthGuard.handleRequest` catches them.
  - `jwt-auth.guard.ts` — `JwtAuthGuard extends AuthGuard('jwt')`. Custom `handleRequest` reads `info.name` and throws `UnauthorizedException('ACCESS_TOKEN_EXPIRED' | 'ACCESS_TOKEN_INVALID' | 'UNAUTHORIZED')` + sets the `WWW-Authenticate` header. **This is how the client (mobile `TokenProvider`) tells reasons for 401 apart**.
  - `admin.guard.ts` — `AdminOnlyGuard implements CanActivate`. Reads `request.user.role`, throws `ForbiddenException` if not `UserRoleEnum.ADMIN`. Used only paired with `JwtAuthGuard`: `@UseGuards(JwtAuthGuard, AdminOnlyGuard)`.
  - `logging.interceptor.ts` — `LoggingInterceptor implements NestInterceptor`. Logs `${method} ${url} ${status} uid=... body=... - ${duration}ms` for POST/PUT, errors — `${method} ${url} ❌ ${err.message}`.
  - `snake-naming.strategy.ts` — TypeORM `NamingStrategyInterface`. Converts camelCase property → snake_case column.
- `src/lib/`:
  - `*.enum.ts` — domain dictionaries (`UserRoleEnum`, `ExperienceEnum`, `GoalPrimaryGoalEnum`, `GoalSecondaryGoalEnum`, `GoalPersonalizationKeyEnum`, `MuscleEnum`, `MuscleGroupEnum`, `EquipmentEnum`, `EquipmentGroupEnum`, `WeightTypeEnum`, `ForceTypeEnum`, `ExerciseCategoryEnum`, `MuscleLoadEnum`). Name — `<Name>Enum`, values — `snake_case` strings. **These enums are duplicated in mobile (domain) and admin intentionally. Any change is breaking for all consumers.**
  - `hash.ts` — `Hash.make(plain)` / `Hash.compare(plain, hash)` (bcryptjs).
- `src/database/`:
  - `database.module.ts` — `@Module({ imports: [ConfigModule.forRoot()], providers: [...dataSourceProviders, ...repositoryProviders, DatabaseService], exports: [...dataSourceProviders, ...repositoryProviders] })`.
  - `database.service.ts` — `typeOrmConfig()` builds `DataSourceOptions`. `entities: [join(__dirname, '..', 'entities', '*.entity.{ts,js}')]`, `migrations: []`, `synchronize: env(POSTGRES_SYNC)`, `namingStrategy: new SnakeNamingStrategy()`. Picks host/port/credentials from `ConfigService`.
  - `database.provider.ts` — `dataSourceProviders` array with one provider `'DATA_SOURCE'`, `useFactory: async (databaseService) => { const ds = new DataSource(databaseService.typeOrmConfig()); if (!ds.isInitialized) await ds.initialize(); return ds; }`.
  - `repository.providers.ts` — `repositoryProviders` array. A single helper `createRepositoryProvider(token, entity)` builds `{ provide: token, useFactory: (ds) => ds.getRepository(entity), inject: ['DATA_SOURCE'] }`. Tokens — `<NAME>_REPOSITORY` UPPERCASE (`'USERS_REPOSITORY'`, `'TRAININGS_REPOSITORY'`, ...). **Every entity that needs a `Repository` must be registered here.**
- `src/i18n/`:
  - `i18n.module.ts` — `@Global() @Module(...)`. Registers `LanguageMiddleware` via `MiddlewareConsumer.apply(LanguageMiddleware).forRoutes('*')`.
  - `language.middleware.ts` — reads `req.headers['accept-language']`, normalizes via `ExerciseExampleI18nService.resolveLanguage(...)`, sets `req.locale: SupportedLanguage`.
  - `i18n.types.ts` — `SUPPORTED_LANGUAGES = ['en', 'ua', 'ru'] as const`, `type SupportedLanguage`, `DEFAULT_LANGUAGE = 'en'`.
  - `locale.helper.ts` — `tryNormalizeLocale(value)` via `Intl.Locale.maximize()`.
  - `express.d.ts` — declaration merge: `interface Request { locale?: SupportedLanguage }`.
  - `exercise-example-i18n.service.ts` — `translateExample(entity, lang)` / `translateExamples(...)` / `translateExercisesCollection(...)` — picks translation by lang, falls back to DEFAULT_LANGUAGE, mutates `entity.name`/`entity.description` and removes `entity.translations` (API contract — no translations array in response).

## Auth flow (end-to-end)

### Endpoints (`AuthController` — `/auth/*`)

- `POST /auth/login` — email + password.
- `POST /auth/register` — email + password (if there is already a Google/Apple user — adds password).
- `POST /auth/google` — Google ID token.
- `POST /auth/apple` — Apple authorization code (inside: `exchangeCodeForIdToken` → `verifyIdToken`).
- `POST /auth/refresh` — `{ refreshToken }` → new access+refresh.

All responses — `LoginResponse { id, accessToken, refreshToken }`.

### Token issuance — `TokenService.createTokens(userId)`

- Access: `JWT_SECRET_KEY`, `JWT_EXPIRATION_TIME`.
- Refresh: `JWT_REFRESH_SECRET`, `JWT_REFRESH_EXPIRATION_TIME` (**`JWT_REFRESH_SECRET` is missing from the README env section — that is a doc gap, not a code gap**).
- Payload — `{ id: userId }`. No roles/email in the token (role is read from DB on every request via `JwtStrategy.validate`).

### Verification — `JwtStrategy`

- `secretOrKey: JWT_SECRET_KEY`, `jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken()`.
- `validate(payload)`:
  1. If `payload.exp * 1000 < Date.now()` — `throw new Error('TokenExpiredError')` (raw, not `UnauthorizedException`).
  2. `usersRepository.findOne({ where: { id: payload.id }, select: ['id', 'email', 'role'] })`. If null — `throw new Error('JsonWebTokenError')`.
  3. Returns `user` → ends up in `request.user`.

### `JwtAuthGuard.handleRequest`

Intercepts the passport result and **translates raw errors into specific 401 codes** + sets `WWW-Authenticate`:

| `info.name` | Response | Header |
|---|---|---|
| `'TokenExpiredError'` | `401 ACCESS_TOKEN_EXPIRED` | `Bearer error="invalid_token", error_description="The access token expired"` |
| `'JsonWebTokenError'` | `401 ACCESS_TOKEN_INVALID` | `Bearer error="invalid_token", error_description="The access token is invalid"` |
| otherwise | `401 UNAUTHORIZED` | `Bearer error="unauthorized", error_description="Authorization required"` |

Mobile `TokenProvider` reads status 401 + `WWW-Authenticate` to decide whether to refresh or log out.

### `AdminOnlyGuard`

Only paired with `@UseGuards(JwtAuthGuard, AdminOnlyGuard)`. Throws `ForbiddenException('Only admin can access this route')` if `user.role !== UserRoleEnum.ADMIN`.

### Refresh contract

`AuthService.refresh(refreshToken)`:

1. `jwtService.verifyAsync<{ id: string }>(refreshToken, { secret: JWT_REFRESH_SECRET })`.
2. `usersRepository.existsBy({ id: payload.id })` — verify the user is not deleted.
3. `tokenService.createTokens(payload.id)` — new pair.
4. On any error — `UnauthorizedException('Invalid or expired refresh token')`.

### Google / Apple flow

`AuthService.loginWithGoogle / loginWithApple`:

1. Verify the ID token (Apple — first exchange code → id token).
2. Lookup user by `googleId` / `appleId`.
3. Fallback lookup by `email` (if there was an email user — link the account).
4. If absent — create with `role: UserRoleEnum.DEFAULT`, `password: null`.
5. If found by email but without `googleId/appleId` — update only those fields.
6. Logs: `New user registered via Google/Apple: ${user.id}` or `User logged in via Google/Apple: ${user.id}`.

## Trainings (full feature cycle)

`TrainingsController` (`/trainings`, `@UseGuards(JwtAuthGuard)`, tag `trainings`):

- `GET /trainings?start=&end=` → `getTrainings(req.user, start, end, req.locale)`.
- `GET /trainings/:id` → `getTrainingById(...)`.
- `POST /trainings` → `createTraining(...)`.
- `PUT /trainings/:id` → `updateTraining(...)`.
- `DELETE /trainings/:id` → `deleteTraining(...)`.

`TrainingsService` notes:

- All methods start with `this.requireProfile(user)` — `BadRequestException` if there's no profile. `getTrainingsByUserId(userId, ...)` (used by `AdminController`) **returns `[]` instead of an exception** if the profile has not been created.
- Date-range queries are validated via `moment(start).isValid()` → `BadRequestException('Wrong date format')` on 400.
- `createQueryBuilder('trainings')` with an explicit chain of `leftJoinAndSelect`:
  - `trainings.exercises` → `exercises.exerciseExample` → `exerciseExample.componentsEntity`, `exerciseExample.translations`, `exercises.iterations`.
  - `orderBy + addOrderBy` by `created_at` ASC, then `order_index` ASC, `created_at` ASC.
- After SQL: manual sort in JS by `orderIndex` and `createdAt` (defensive). Then for each `training.exercises`:
  1. `this.exerciseExampleI18nService.translateExercisesCollection(training.exercises, language)` — picks translation, mutates `name`/`description`, removes `translations`.
  2. `this.attachComponentsToExerciseExample(exercise.exerciseExample)` — converts `componentsEntity` into a public `components: ExerciseComponentsDto` (mutating add + delete `componentsEntity`).
- `createTraining` / `updateTraining` are wrapped in `this.trainingsRepository.manager.transaction(async (manager) => { ... })`. Inside:
  - `manager.create(<Entity>, { ...rest, id: v4() })` for each entity (id explicit via uuid).
  - `update`: first `manager.delete(ExercisesEntity, { trainingId })` (cascade removes iterations), then re-insert.
  - Each `exerciseExampleId` is validated via `exerciseExamplesRepository.findOneBy(...)` → `BadRequestException('Invalid exerciseExampleId: ...')` for dangling references.
- DELETE checks `findOne({ where: { id, profileId: profile.id } })` → `NotFoundException(\`Training with id ${id} not found or access denied\`)`.

### DTO patterns

`TrainingsRequest` (POST/PUT body):

```typescript
export class TrainingsRequest {
  @ApiProperty({ type: 'number', example: 25, description: 'Duration in minutes' })
  @IsInt()
  duration: number;

  @ApiProperty({ type: 'number', example: 20000.5, description: 'Total volume in kg (1 decimal)' })
  @IsNumber({ maxDecimalPlaces: 1 })
  volume: number;

  @ApiProperty({ type: [TrainingExerciseRequest] })
  @ValidateNested({ each: true })
  @Type(() => TrainingExerciseRequest)
  exercises: TrainingExerciseRequest[];
}
```

`TrainingCreateResponse` — minimal output:

```typescript
export class TrainingCreateResponse {
  @ApiProperty({ type: 'string', example: '...', description: 'Training ID' })
  @IsUUID()
  id: string;
}
```

GET endpoints return `TrainingsEntity` directly (typed in Swagger via `@ApiOkResponse({ type: [TrainingsEntity] })` + `@ApiExtraModels(...)`). This is an **intentional** trade-off — entity with `transformer`s on decimals and without `select: false` fields acts as the response shape. If the entity gains sensitive fields — they must use `select: false` (like `password` in `UsersEntity`).

### Entities (typical pattern)

```typescript
@Entity({ name: 'trainings' })
export class TrainingsEntity {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({ type: 'decimal', precision: 10, scale: 1, nullable: true,
    transformer: { to: (v: number) => v, from: (v: string) => parseFloat(v) }})
  volume: number | null;

  @Column({ name: 'profile_id', type: 'uuid', nullable: true })
  profileId: string | null;

  @Index()
  @CreateDateColumn({ type: 'timestamp without time zone', name: 'created_at' })
  createdAt: Date;

  @UpdateDateColumn({ type: 'timestamp without time zone', name: 'updated_at' })
  updatedAt: Date;

  @OneToMany(() => ExercisesEntity, (e) => e.training, { cascade: ['remove'] })
  exercises: ExercisesEntity[];

  @ManyToOne(() => UserProfilesEntity, (p) => p.trainings,
    { onDelete: 'CASCADE', orphanedRowAction: 'delete' })
  @JoinColumn({ name: 'profile_id' })
  profile: UserProfilesEntity;
}
```

Conventions:

- `@PrimaryGeneratedColumn('uuid')` for all primary keys (but in code `id` is usually generated by hand via `v4()` in service for transaction control).
- `decimal` columns — `transformer` to coerce a PostgreSQL string into a JS number.
- `timestamp without time zone` for `createdAt`/`updatedAt` (UTC).
- FK `@ManyToOne` always with explicit `onDelete` (`'CASCADE'` for child→parent dependencies, `'SET NULL'` where the child should outlive the parent).
- `cascade: ['remove']` on `@OneToMany` for auto-removal of children.
- `orphanedRowAction: 'delete'` where appropriate.
- `@JoinColumn({ name: 'snake_case' })` — explicitly.
- `@Index()` on FK fields and on fields used in frequent filters.

## Validation reference (class-validator)

Every DTO field **must** be annotated with validators. The global
`ValidationPipe({ whitelist: true, forbidNonWhitelisted: true, transform: true })`
means:

- `whitelist` — fields without decorators are **discarded**.
- `forbidNonWhitelisted` — an extra field in the request body → **400**.
- `transform` — `class-transformer` coerces primitives (`@Type(() => Number)`).

Decorators in use (use only from this list; new ones require discussion):

| Decorator | Use |
|---|---|
| `@IsString()` | String. |
| `@IsInt()` | Integer (no decimals). |
| `@IsNumber({ maxDecimalPlaces: 1 })` | Number with precision limit. |
| `@IsUUID()` / `@IsUUID('all', { each: true })` | UUID for one / array. |
| `@IsEmail()` | Email. |
| `@IsEnum(<Enum>)` | Enum value. |
| `@Length(min, max)` | String length. |
| `@IsOptional()` | Field may be absent. Pairs with `nullable` in `@ApiProperty`. |
| `@ValidateNested({ each: true })` + `@Type(() => Nested)` | Nested DTOs. **Without `@Type`, the validator does not run on nested.** |
| `@Type(() => String)` | Forced conversion (for an array of strings). |
| `@Expose()` | `class-transformer` — preserve a field after `plainToInstance`. |

Pattern for an optional nullable number:

```typescript
@ApiProperty({ type: 'number', example: 20, nullable: true, required: false })
@IsNumber({ maxDecimalPlaces: 1 })
@IsOptional()
extraWeight?: number | null;
```

Pattern for a nested array:

```typescript
@ApiProperty({ type: [TrainingExerciseRequest] })
@ValidateNested({ each: true })
@Type(() => TrainingExerciseRequest)
exercises: TrainingExerciseRequest[];
```

Pattern for an optional UUID array:

```typescript
@ApiProperty({ example: ['...'], type: 'array', required: false })
@IsUUID('all', { each: true })
@IsOptional()
@Type(() => String)
@Expose()
excludeMuscleIds?: string[];
```

`AdminModule` — single `AdminController` for all admin endpoints:

```typescript
@Controller('admin')
```

Imports other feature modules (`ExerciseExampleModule`, `UsersModule`, `TrainingsModule`, `WeightHistoryModule`) and reuses their services. **Not split** — intentionally one controller for all `/admin/*`.

Route groups inside `AdminController`:

- `POST/PUT/DELETE /admin/exercise-examples` — CRUD for exercise examples.

Patterns for admin routes:

- `@Param('id', new ParseUUIDPipe())` for UUID validation of path parameters.

- `@HttpCode(HttpStatus.NO_CONTENT)` for destructive ops without a response body.

## Push tokens

A separate `PushTokensModule` with one table. Upsert pattern:

```typescript
async upsertToken(userId: string, token: string): Promise<void> {
  await this.pushTokensRepository.createQueryBuilder()
    .insert().into(PushTokensEntity)
    .values({ user: { id: userId }, token })
    .orUpdate(['user_id'], ['token'])
    .execute();
}
```

— upsert via `orUpdate(['<conflict-key>'], ['<update-key>'])`, not two queries.

## i18n details

`ExerciseExampleI18nService.translateExample(entity, language)`:

1. `translations = entity.translations ?? []`.
2. Look up `directTranslation = translations.find(t => t.language === language)`.
3. Fallback `defaultTranslation = translations.find(t => t.language === DEFAULT_LANGUAGE)`.
4. `selected = directTranslation ?? defaultTranslation ?? null`.
5. If present — mutate `entity.name`/`entity.description` (with fallback to the originals).
6. `delete (entity as any).translations` — the translations array **never goes into the response**.

`translateExercisesCollection(exercises, lang)` does the same for each `exercise.exerciseExample`.

`resolveLanguage(header)`:

- Parses `Accept-Language` via `tryNormalizeLocale` (uses `Intl.Locale.maximize()`).
- Returns the first language from the header that fits into `SUPPORTED_LANGUAGES`. Otherwise `DEFAULT_LANGUAGE = 'en'`.

In controllers:

```typescript
const language = req.locale ?? this.exerciseExampleI18nService.resolveLanguage();
return this.trainingsService.getTrainings(req.user, start, end, language);
```

`req.locale` is always populated thanks to `LanguageMiddleware` globally, but the service fallback is defensive.

## Code style and naming

### Files and naming

- Files — kebab-case (`auth.service.ts`, `user-profiles.entity.ts`, `create-user-profile.request.ts`).
- Classes — PascalCase, methods/fields — camelCase, env vars — `UPPER_SNAKE_CASE`, repo tokens — `'<NAME>_REPOSITORY'` (UPPER_SNAKE_CASE string).
- One class per file, **no `export default`**.
- Imports — single `{ ... }`, no default, no barrel files.
- Suffix conventions:

| Type | File | Class |
|---|---|---|
| Module | `<feature>.module.ts` | `<Name>Module` |
| Service | `<feature>.service.ts` | `<Name>Service` |
| Controller | `<feature>.controller.ts` | `<Name>Controller` |
| DTO request | `dto/<name>.request.ts` | `<Name>Request` |
| DTO response | `dto/<name>.response.ts` | `<Name>Response` |
| Sub-DTO | `dto/<name>.dto.ts` | `<Specific><Parent>Dto` |
| Entity | `<name>.entity.ts` | `<Name>Entity` |
| Enum | `lib/<name>.enum.ts` | `<Name>Enum` |
| Strategy | `common/<name>.strategy.ts` | `<Name>Strategy` |
| Guard | `common/<name>.guard.ts` | `<Name>Guard` (exception: `AdminOnlyGuard`) |
| Interceptor | `common/<name>.interceptor.ts` | `<Name>Interceptor` |
| Middleware | `i18n/<name>.middleware.ts` | `<Name>Middleware` |

### NestJS conventions

- Global `ValidationPipe` with `forbidNonWhitelisted: true` — **any extra field in the request → 400**. So DTOs do not contain `extends` wrappers with extra fields.
- Every endpoint:
  - `@HttpCode(HttpStatus.OK | CREATED | NO_CONTENT)` — explicitly (Nest defaults POST → 201, others → 200).
  - `@ApiOperation({ summary: '...' })` — for Swagger.
  - `@ApiResponse({ status, description, type })` — for every expected outcome.
  - `@ApiBearerAuth('access-token')` on controllers with `@UseGuards(JwtAuthGuard)` (not on every method).
  - `@ApiTags('<feature>')` on the controller.
  - `@ApiExtraModels(<Entity>, <Dto>, ...)` for types referenced via `type: [<Entity>]` without an explicit `@ApiProperty(... type: () => Entity)`.
- Errors — **only NestJS HttpExceptions** (`UnauthorizedException`, `BadRequestException`, `ForbiddenException`, `NotFoundException`, `ConflictException`). Do not throw `new Error('...')` (exception — `JwtStrategy.validate`, where a raw `Error` is needed for `JwtAuthGuard.handleRequest`).
- Services hold `private readonly logger = new Logger(<Service>.name)`. Logging:
  - `this.logger.log(...)` — info (`User logged in: ${user.id}`).
  - `this.logger.warn(...)` — recoverable (`Login failed for ${dto.email}`).
  - `this.logger.error(...)` — exceptions.
- DI repositories — `@Inject('<NAME>_REPOSITORY') private readonly <name>Repository: Repository<<Name>Entity>`. **No `@InjectRepository()` from `@nestjs/typeorm`** — we have our own `repository.providers.ts`.
- No `@nestjs/typeorm` module, no `forFeature([...])`. Only our `DatabaseModule + repositoryProviders`.
- `@Req()` / `@Res({ passthrough: true })` — typed. `@Req() req: Request` (Express). `req.user.id` is available after `JwtAuthGuard`.

### TypeScript

- `tsconfig.json` — `"strict": true`. That said, `req: any`/`user: any` shows up where `Request` is not augmented (admin DTO, controllers). For new signatures — type properly.
- `class-validator` decorators are paired with `class-transformer` (`@Type(() => X)` for nested, `@Expose()` for preservation after plain-to-class).
- Date-range params — strings ISO, validate with `moment(value).isValid()` or regex; never `new Date(...)` without a check.

### Service patterns

- **Transactions**: `await this.<entity>Repository.manager.transaction(async (manager) => { ... })`. Inside, fetch repos via `manager.getRepository(<Entity>)` — **do not use injected repos** (they live outside the transaction).
- **Range queries**: `createQueryBuilder` with `.where(...).andWhere(...)`, FK joins via `leftJoinAndSelect`, sort via `.orderBy(...).addOrderBy(...)`. `.getMany()` / `.getOne()` / `.getRawMany<Shape>()` for aggregates.
- **Sort after SQL**: for nested collections (`exercise.iterations`) re-sort in JS — TypeORM does not guarantee order with relations.
- **Authorization**: every user-scoped query starts with `requireProfile(user)` (or `usersRepository.findOne({ where: { id: user.id } })`). Access is enforced via `where: { id, profileId: profile.id }`. On miss — `NotFoundException(\`<Entity> with id ${id} not found or access denied\`)`.
- **Aggregations** (`getRawMany`): group via `.groupBy(...)`, `Number(row.field) || 0` for coercion, `Map<id, value>` for O(1) lookup during post-processing.
- **Conflict handling**: `findOneBy` → `ConflictException(...)` or `BadRequestException(...)` if the value is already taken.

## Performance budgets and priorities

No explicit budgets in code. Defaults:

- Don't N+1. Use `relations: [...]` or `createQueryBuilder` with `leftJoinAndSelect`.
- List endpoints — must have either pagination or a date-range filter (`/trainings?start=&end=`, `/admin/users/:id/trainings?start=&end=`, `/admin/users/:id/weight-history`).
- Use aggregation queries (`getRawMany`) instead of JS-side reduces.
- Don't block the event loop with heavy synchronous work (bcrypt — sync `compareSync`/`hashSync`, but one call per login → acceptable).

## Locked architectural decisions

- TypeORM `synchronize` instead of migrations — intentional. `migrations: []`, `migrationsRun` controlled by env. **Do not introduce `migrations: [...]` without a request.**
- NestJS modules-by-feature, not CQRS, not event-driven, not GraphQL.
- Single `AdminController` for all `/admin/*` — intentional, do not split.
- JWT in Bearer header, **not cookies**.
- Global `ValidationPipe` with strict `forbidNonWhitelisted` — any extra field in request → 400.
- Custom `repositoryProviders` instead of `@nestjs/typeorm.forFeature()` — uniform DI style via `'<NAME>_REPOSITORY'` tokens.
- Custom `JwtAuthGuard.handleRequest` maps errors to specific codes (`ACCESS_TOKEN_EXPIRED`/`ACCESS_TOKEN_INVALID`/`UNAUTHORIZED`) — clients depend on these strings.
- Refresh — separate secret (`JWT_REFRESH_SECRET`) and TTL (`JWT_REFRESH_EXPIRATION_TIME`). Payload only `{ id }`.
- ID — `uuid` v4 (generated by hand in service via `v4()`).
- Docker Compose as the single way to run (db + backend in one network).
- `synchronize` controlled by env (`POSTGRES_SYNC`), enabled in prod — intentional.
- I18n module is `@Global()` — `LanguageMiddleware` everywhere, `req.locale` is available in any controller.

## Scope discipline

- Do not add migrations (`migrations: [...]`, `migrationsRun: true`, a new migrations folder) without a request. Schema is rolled forward via `synchronize`.
- Do not edit `entities/*.entity.ts` without a request — this is the production schema controlled by `synchronize`. Any `non-nullable` column without a default added to an existing table will break the deploy.
- Do not edit `DatabaseService.typeOrmConfig()` without a request.
- Do not edit `repository.providers.ts` (order/tokens) without a request.
- Do not edit `Dockerfile` / `docker-compose.yml` / `scripts/deploy.sh` without a request.
- Do not introduce cookies / sessions / a different auth mechanism.
- Do not change `JwtAuthGuard.handleRequest` error codes (`ACCESS_TOKEN_EXPIRED`/`ACCESS_TOKEN_INVALID`) — mobile `TokenProvider` depends on them.
- Do not add global guards/interceptors in `main.ts` without a request.
- Do not change the `.env` structure (all env vars are documented in `README.md`).
- Do not introduce `@nestjs/typeorm` `forFeature(...)` — we have our own DI.
- Do not change `ValidationPipe` parameters (`forbidNonWhitelisted` is critical for security).

## When to stop and ask

- Any DB schema change (new column, type change, removal, index, change of `nullable`/`default`).
- Any new endpoint in `/admin/*` or change to `JwtAuthGuard`/`AdminOnlyGuard`/`JwtStrategy`/`TokenService`.
- Change to a DTO of an existing endpoint — **API breaking change**, sync with mobile (`grippo-mobile/data-services/backend/dto/...`) and admin (`grippo-admin/src/infrastructure/http/endpoints.js`).
- Change to an enum in `src/lib/*.enum.ts` — breaking for all consumers.
- Change to Auth flow (Google/Apple/refresh/login), Token TTL, `JWT_*` env vars.
- New dependency in `package.json`.
- Any major rework in `users.service.ts` or `trainings.service.ts` (transactions, query patterns) — both contain access checks and cascading deletes.
- New OAuth provider (new column in `UsersEntity`, new service, new flow).
- Change to `setupSwagger` — Swagger is consumed by external systems.

## Anti-patterns (refuse to write)

- Business logic in controllers. Controller — only HTTP + DTO + `@UseGuards` + Swagger decorators + service call.
- Direct `getRepository(Entity)` or static access to `DataSource`. Only injection via `@Inject('<NAME>_REPOSITORY')` or (inside a transaction) `manager.getRepository(...)`.
- `@nestjs/typeorm` `@InjectRepository()` / `TypeOrmModule.forFeature(...)` — we have our own DI via `repository.providers.ts`.
- Returning a TypeORM entity directly if it has `@Column({ select: false })` fields (like `password`). Use a response DTO.
- Inline SQL without `QueryBuilder` (with the exception of raw migrations, which we don't have).
- Any locale work outside `LanguageMiddleware` / `req.locale` / `ExerciseExampleI18nService`.
- Generic `throw new Error('...')` in business logic. Only NestJS `HttpException`s. (Exception — `JwtStrategy.validate`, raw error for `JwtAuthGuard`.)
- Date arithmetic via `new Date(...)` without input validation. Use `moment(input).isValid()`.
- Cross-feature import of **services** directly without `imports: [<Other>Module]`. NestJS won't allow it without exports/imports, but someone might try via a relative import — don't.
- Transactions via an `await` chain without `manager.transaction(...)` — there will be a race in `INSERT` of child entities. Inside a transaction, **always** use `manager.getRepository()`, not the injected repo.
- Global mutable modules. State lives in DB.
- `process.env.X` directly — only via `ConfigService.get<...>('X')`.
- Direct injection of `Request`/`Response` Express objects beyond edge cases — prefer `@Req()`/`@Res({ passthrough: true })` with typing.
- DTOs without `@ApiProperty` — Swagger won't pick them up, mobile won't see the contract.
- DTOs without at least one `class-validator` decorator on fields — `whitelist` will drop the field from the request body.
- Custom nested DTOs without `@ValidateNested({ each: true }) + @Type(() => Nested)` — the validator will skip nested.
- `console.log` — only `Logger`. NestJS `Logger` integrates with morgan and formats uniformly.
- `@Module({ controllers: [...], providers: [...] })` without `imports: [DatabaseModule]` if you use `repositoryProviders` — you'll get `Nest can't resolve dependencies`.
- Touching `password` in a response. `@Column({ select: false })` already won't pick it up — but never return it via a manual `findOne({ select: ['id', 'password'] })` in a controller.
